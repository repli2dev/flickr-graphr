Uživatelské rozhraní

Při návrhu projektu byly zvoleny technologie HTML + JavaScript, které umožňují vyšší interaktivitu než staticky generovaný obsah. Uživatelské rozhraní komunikuje se serverovou částí pomocí REST API, data jsou posílána pomocí JSON objektů.

Z důvodu security policy mají prohlížeče zakázány dotazy na bázi XmlHttpRequest na jiný server, než ze kterého je generovaná statická stránka, proto musí být UI i serverová část přítomna na jednom serveru. Toto omezení by bylo možné vyřešit pomocí technologie JSONP, ovšem za cenu snížení možnosti reagovat na případné na straně serveru.

Webové rozhraní se dělí do jednotlivých modulů (users, tags, photos), kde každý modul obsahuje jednu vizualizaci. Moduly se načítají asynchronně kliknutím na odkaz v menu, v budoucnu lze moduly upravovat i přidávat moduly nové.

Modul se skládá ze statické stránky (template.html) a ze souboru se skripty (script.js), konkrétní struktura je popsaná ve Vývojářské dokumentaci ke GUI.

jQuery, jQuery UI, pluginy HashChange a Tagcloud

jQuery, jeden z nejrozšířenějších JavaScript frameworků, zajišťuje načítání jednotlivých modulů webového rozhraní a prostředky pro komunikaci s REST API. jQuery UI poskytuje připravené základní prvky uživatelského rozhraní (date picker, tlačítko, ...). Všechny použité pluginy do jQuery jsou volně šířitelné.

Plugin HashChange umožňuje jednodušší obsluhování události změny URL, používá se při načítání modulů, Tagcloud generuje objekt se štítky, u kterého je velikost názvu štítku dána počtem jeho výskytů.

Google Charts

Ze široké nabídky vizualizací je použit interaktivní Line Chart, pro který jsou data získaná z REST API speciálně upravena (API data posílá v objektech, ze kterých je generovaná tabulka - zdroj pro Charts). Přidáním položky do grafu se graf znovu vykreslí.

Flickr API

Flickr API je použito přímo i v UI. Kliknutím na položky v některých grafech jsou načítány doplňující informace (uživatelé, fotografie) přímo z Flickru.

Shrnutí

Projekt Flickr-graphr byl rozdělen do několika částí, kde každou měl přidělen některý z členů týmu. To se osvědčilo. Výsledkem je funkční, uživatelsky přívětivá, webová aplikace, která má šanci uplatnit se v praxi.
